## [1011 Fly me to the Alpha Centauri](https://www.acmicpc.net/problem/1011)

### 문제

우현이는 어린 시절, 지구 외의 다른 행성에서도 인류들이 살아갈 수 있는 미래가 오리라 믿었다. 그리고 그가 지구라는 세상에 발을 내려 놓은 지 23년이 지난 지금, 세계 최연소 ASNA 우주 비행사가 되어 새로운 세계에 발을 내려 놓는 영광의 순간을 기다리고 있다.

그가 탑승하게 될 우주선은 Alpha Centauri라는 새로운 인류의 보금자리를 개척하기 위한 대규모 생활 유지 시스템을 탑재하고 있기 때문에, 그 크기와 질량이 엄청난 이유로 최신기술력을 총 동원하여 개발한 공간이동 장치를 탑재하였다. 하지만 이 공간이동 장치는 이동 거리를 급격하게 늘릴 경우 기계에 심각한 결함이 발생하는 단점이 있어서, 이전 작동시기에 k광년을 이동하였을 때는 k-1 , k 혹은 k+1 광년만을 다시 이동할 수 있다. 예를 들어, 이 장치를 처음 작동시킬 경우 -1 , 0 , 1 광년을 이론상 이동할 수 있으나 사실상 음수 혹은 0 거리만큼의 이동은 의미가 없으므로 1 광년을 이동할 수 있으며, 그 다음에는 0 , 1 , 2 광년을 이동할 수 있는 것이다. ( 여기서 다시 2광년을 이동한다면 다음 시기엔 1, 2, 3 광년을 이동할 수 있다. )


김우현은 공간이동 장치 작동시의 에너지 소모가 크다는 점을 잘 알고 있기 때문에 x지점에서 y지점을 향해 최소한의 작동 횟수로 이동하려 한다. 하지만 y지점에 도착해서도 공간 이동장치의 안전성을 위하여 y지점에 도착하기 바로 직전의 이동거리는 반드시 1광년으로 하려 한다.

김우현을 위해 x지점부터 정확히 y지점으로 이동하는데 필요한 공간 이동 장치 작동 횟수의 최솟값을 구하는 프로그램을 작성하라.

### 입력

입력의 첫 줄에는 테스트케이스의 개수 T가 주어진다. 각각의 테스트 케이스에 대해 현재 위치 x 와 목표 위치 y 가 정수로 주어지며, x는 항상 y보다 작은 값을 갖는다. ( 0 ≤ x < y < 231)

### 출력

각 테스트 케이스에 대해 x지점으로부터 y지점까지 정확히 도달하는데 필요한 최소한의 공간이동 장치 작동 회수를 출력한다.

## 풀이

처음에 생각보다 쉬운문제라고 생각해서 규칙만 찾으면 끝인줄 알았다.

첫번째 워프와 마지막 워프는 1이여야 하고, 중간부터의 워프 횟수는 `k-1, k, k+1` 이기 때문에 거리를 1부터 쭉 나열하면 규칙을 찾을 수 있을거라 생각했기 때문이다.

하지만 큰 오산이였다.

| 차수 | 수열 | 거리 | 워프횟수 | 최고속도 |
|:-----|:----|:-----|:--------|:--------|
| 1 | 1 | 1 | 1 | 1 |
| 2 | 121 | 4 | 3 | 2 |
| 3 | 12321 | 9 | 5 | 3 |
| 4 | 1234321 | 16 | 7 | 4 |
| ... | ... | ... | ... | ... |
| n | 1..n..1 | n^2 | 2n-1 | n |

거리를 기준으로 손으로 쭉 수열을 나열해보면 규칙을 찾을수 있게 된다.

하지만 17이나 23같은 나머지 거리를 구하는데 고민을 하고 있었는데 하나의 힌트글을 보게 되었다. (정답수준의 힌트)

[힌트](https://www.acmicpc.net/board/view/13747)

즉 남은거리는 최고속도 보다 빨리 갈수 없고, 최고속도로 나눠서 올림하면 추가로 가야하는 횟수가 나온다는 것이다.

예를 들어 23의 경우

16보다 크고 25보다 작기때문에 거리는 16, 최고속도는 4이며 워프횟수는 7(2*4-1)이다.

나머지 거리는 23-16 으로 7이 나오는데 이때 7을 4로 나누면 1.75 즉 2회 더 가야한다는 결과가 나온다.

```
거리 : 23

1234321 = 16
최고속도 4 이하의 수를 2회 삽입
123 (3) 4 (4) 321 = 23
```

### Code

```cs
using System;

namespace ConsoleApp1 {
    class Program {
        static void Main() {
            int n_count = int.Parse(Console.ReadLine());

            for(int n_idx = 0; n_idx < n_count; n_idx++) {
                string[] values = Console.ReadLine().Split(' ');
                long n_dist = long.Parse(values[1]) - long.Parse(values[0]);

                long n_factor = 1;

                while(true) {
                    if (n_factor * n_factor > n_dist)
                        break;
                    n_factor++;
                }

                n_factor -= 1;

                long n_remain = n_dist - (n_factor * n_factor);
                n_remain = (long)Math.Ceiling((double)n_remain / n_factor);

                Console.WriteLine((n_factor * 2) - 1 + n_remain);
            }
        }
    }
}
```

## 후기

솔직히 저 힌트글을 보고 천재라고 생각했다. 어떻게 저런 공식을 구할 수 있었을까 나도 나중에 저렇게 공식을 도출할 수 있을까? 그렇게 하려면 어떤 노력을 해야할까 이런 저런 생각이 드는 문제였다. 솔직히 저 글을 보지 못했어도 못 풀 정도의 문제는 아니였다. 어느정도 규칙을 찾은 상태여서 이후의 규칙에 대해 나름대로 풀었으면 될 것 같았는데 저 힌트글을 본 순간 너무 깔끔하게 값이 나와서 따로 풀어보진 않았다...

참고로 n_dist, n_factor 같이 사용되는 정수형 변수들을 int로 하면 런타임 에러가 뜬다. 굉장히 큰 값까지 테스트케이스로 존재하는 모양인데 저것때문에 2시간정도를 헤맸다.
